   1  This files describes API changes in the assign code.
   2  
   3  === 3.2 ===
   4  * External function mod_assign_external::get_assignments now returns additional optional fields:
   5     - preventsubmissionnotingroup: Prevent submission not in group.
   6     - submissionstatement and submissionstatementformat: When there is a submission statement defined.
   7  * Proper checking for empty submissions
   8  * Submission modification time checking - this will help students working in groups not clobber each others'
   9    submissions
  10  * External functions that were returning file information now return the following file fields:
  11    filename, filepath, mimetype, filesize, timemodified and fileurl.
  12    Those fields are now marked as VALUE_OPTIONAL for backwards compatibility.
  13    Please, note that previously the filename was part of the filepath field, now they are separated.
  14  
  15  === 3.1 ===
  16  * The feedback plugins now need to implement the is_feedback_modified() method. The default is to return true
  17    for backwards compatibiltiy.
  18  * When downloading all submissions as a zip each students' files are in a separate folder, are no longer renamed
  19    and the folder structure is kept intact.
  20  
  21  === 3.0 ===
  22  * assign_submission_status renderable now requires $usergroups in its constructor
  23  * External function mod_assign_external::get_assignments now have a new optional parameter: includenotenrolledcourses.
  24    Its default value is false, setting it to true will return assignments for courses a user can see even if it's not enroled.
  25  
  26  === 2.9 ===
  27  * External parameter now returns PARAM_RAW instead of PARAM_TEXT for webservices using feedback comments.
  28  * External function mod_assign_external::get_assignments now returns additional optional fields:
  29     - intro: Assignment intro.
  30     - introformat: Intro format.
  31     - introattachments: Intro attachments files.
  32  
  33  === 2.8 ===
  34  * Some DB changes were made to simplify the SQL required to query the latest attempt.
  35    - The assign_submission table now has a column "latest" which is set to 1 for the latest submission attempt.
  36    - There will always be a submission row if there is a grade (so the latest grade can be found by joining with the submission)
  37    - There is a new submission status "new" for a submission that has never been attempted by a student (but the record exists purely
  38      to mark the latest attempt number as 0). The function get_user_submission will create a record with the status set to "new"
  39      by default (the previous default was "draft").
  40  
  41  === 2.7 ===
  42  
  43  * Added setting sendstudentnotifications to assign DB table with admin defaults. This sets the default value for the
  44    "Notify students" option on the grading forms. This setting can be retrieved via webservices.
  45  
  46  * Web service function mod_assign_save_grade has an additional optional parameter $advancedgradingdata which allows
  47    advanced grading data to be used.
  48  * A new web service function mod_assign_save_grades has been added which allows multiple grades to be processed.
  49  * The event submission_updated() [mod/assign/classes/event/submission_updated.php] has been altered to now be an abstract class
  50    for submission events in the submission plug-ins.
  51  
  52  === 2.6.1 ===
  53  
  54  * format_text() is no longer used for formating assignment content to be used in events (assign_submission_onlinetext::save()) or
  55    the word count (assign_submission_onlinetext::format_for_log()) in mod/assign/submission/onlinetext/locallib.php. format_text()
  56    should only be used when displaying information to the screen. It was being used incorrectly before in these areas. Plugins using
  57    the event assessable_uploaded() should use file_rewrite_pluginfile_urls() to translate the text back to the desired output.
  58  
  59  === 2.6 ===
  60  * To see submission/grades of inactive users, user should have moodle/course:viewsuspendedusers capability.
  61  * count_* functions will return only active participants.
  62  * assign_submission_plugin->lock and unlock methods have an additional parameter for user flags. A user will not
  63    always have a submission record when the submission is locked/unlocked.
  64  * Submission and feedback plugins can now participate in webservices. The plugin must implement get_external_parameters()
  65    to describe the parameters it is expecting from the mod_assign_save_grade or mod_assign_save_submission functions. The
  66    plugin will then handle the webservice via it's normal save() method with the extra data supplied in the $data argument.
  67  
  68  === 2.5 ===
  69  
  70  * New feature - Attempt History
  71  
  72      This adds settings so that a student can have build up a history of separate submission attempts and grades for the same
  73      assignment.
  74  
  75      Extra settings on the assign table are:
  76      attemptreopenmethod - The way attempts can be reopened. One of:
  77                              ASSIGN_ATTEMPT_REOPEN_METHOD_NONE - Attempts cannot be reopened (default)
  78                              ASSIGN_ATTEMPT_REOPEN_METHOD_UNTIL_PASS - Attempts are reopened on grading until the gradebook
  79                                                                        indicates this student has passed.
  80                              ASSIGN_ATTEMPT_REOPEN_METHOD_MANUAL - Attempts are reopened on manually by the grader.
  81      maxattempts - The maximum allowed number of attempts per student for this assign instance.
  82  
  83      Extra settings on the submission and grade records:
  84      attemptnumber - Starting from 0 until the latest attempt.
  85  
  86      New table assign_user_flags holds user specific flags that were stored in the assign_grade table.  (contains assignment,
  87      userid, extensionduedate, mailed and locked)
  88  
  89      assign changes:
  90          "get_user_grade" function has an extra optional parameter to retrieve a specific attempt number.
  91          "get_user_submission" function has an extra optional parameter to retrieve a specific attempt number.
  92          "get_group_submission" function has an extra optional parameter to retrieve a specific attempt number.
  93          new "get_user_flags" function can retrieve the user flags for a specific user (extensionduedate, mailed and locked).
  94          new "update_user_flags" function can update the user flags for a specific user (extensionduedate, mailed and locked).
  95  
  96      assign_submission_plugin changes:
  97          "precheck_submission" function now takes a submission record so you can determine which submission is being checked.
  98          "submit_for_grading" function now takes a submission record so you can determine which submission is being submitted.
  99          new function "copy_submission" can be implemented to copy submission data from one submission to a new one.
 100  
 101      New renderable object "assign_attempt_history" for rendering the list of previous submissions.
 102      New renderable object "assign_gradingmessage" for rendering a generic grading message.
 103  
 104      assign_grading_table changes:
 105          get_group_and_submission is now protected and should not be called from outside this class.
 106  
 107  
 108  === Earlier changes ===
 109  
 110  * Were not documented in this way. Sorry.
