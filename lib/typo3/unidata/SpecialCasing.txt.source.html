   4  # Special Casing Properties
   5  #
   6  # This file is a supplement to the UnicodeData file.
   7  # It contains additional information about the casing of Unicode characters.
   8  # (For compatibility, the UnicodeData.txt file only contains case mappings for
   9  # characters where they are 1-1, and does not have locale-specific mappings.)
  10  # For more information, see the discussion of Case Mappings in the Unicode Standard.
  11  #
  12  # All code points not listed in this file that do not have a simple case mappings
  13  # in UnicodeData.txt map to themselves.
  14  # ================================================================================
  15  # Format
  16  # ================================================================================
  17  # The entries in this file are in the following machine-readable format:
  18  #
  19  # <code>; <lower> ; <title> ; <upper> ; (<condition_list> ;)? # <comment>
  20  #
  21  # <code>, <lower>, <title>, and <upper> provide character values in hex. If there is more than
  22  # one character, they are separated by spaces. Other than as used to separate elements,
  23  # spaces are to be ignored.
  24  #
  25  # The <condition_list> is optional. Where present, it consists of one or more locales or contexts,
  26  # separated by spaces. In these conditions:
  27  # - A condition list overrides the normal behavior if all of the listed conditions are true.
  28  # - The context is always the context of the characters in the original string,
  29  #   NOT in the resulting string.
  30  # - Case distinctions in the condition list are not significant.
  31  # - Conditions preceded by "Not_" represent the negation of the condition.
  32  #
  33  # A locale is defined as:
  34  # <locale> := <ISO_639_code> ( "_" <ISO_3166_code> ( "_" <variant> )? )?
  35  # <ISO_3166_code> := 2-letter ISO country code,
  36  # <ISO_639_code> :=  2-letter ISO language code
  37  #
  38  # A context is one of the following, as defined in the Unicode Standard:
  39  #   Final_Sigma, After_Soft_Dotted, More_Above, Before_Dot, Not_Before_Dot, After_I
  40  #
  41  # Parsers of this file must be prepared to deal with future additions to this format:
  42  #  * Additional contexts
  43  #  * Additional fields
  44  # ================================================================================
  45
