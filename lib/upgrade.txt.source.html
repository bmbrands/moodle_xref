   1  This files describes API changes in core libraries and APIs,
   2  information provided here is intended especially for developers.
   3  
   4  === 2.9 ===
   5  
   6  * The default home page for users has been changed to the dashboard (formely my home). See MDL-45774.
   7  * Support for rendering templates from php or javascript has been added. See MDL-49152.
   8  * Support for loading AMD javascript modules has been added. See MDL-49046.
   9  * Webservice core_course_delete_courses now return warning messages on any failures and does not try to rollback the entire deletion.
  10  * \core\event\course_viewed 'other' argument renamed from coursesectionid to coursesectionnumber as it contains the section number.
  11  * New API core_filetypes::add_type (etc.) allows custom filetypes to be added and modified.
  12  * PHPUnit: PHPMailer Sink is now started for all tests and is setup within the phpunit wrapper for advanced tests.
  13    Catching debugging messages when sending mail will no longer work. Use $sink = $this->redirectEmails(); and then check
  14    the message in the sink instead.
  15  * The file pluginlib.php was deprecated since 2.6 and has now been removed, do not include or require it.
  16  * \core_component::fetch_subsystems() now returns a valid path for completion component instead of null.
  17  * Deprecated JS global methods have been removed (show_item, destroy_item, hide_item, addonload, getElementsByTagName, findChildNodes).
  18  * For 3rd party plugin specific environment.xml files, it's now possible to specify version independent checks by using the
  19    <PLUGIN name="component_name"> tag instead of the version dependent <MOODLE version="x.y"> one. If the PLUGIN tag is used any
  20    Moodle specific tags will be ignored.
  21  * html_table: new API for adding captions to tables (new field, $table->caption) and subsequently hiding said captions from sighted users using accesshide (enabled using $table->captionhide).
  22  * The authorization procedure in the mdeploy.php script has been improved. The script
  23    now relies on the main config.php when deploying an available update.
  24  * sql_internal_reader and sql_select_reader interfaces have been deprecated in favour of sql_internal_table_reader
  25    and sql_reader which use iterators to be more memory efficient.
  26  * $CFG->enabletgzbackups setting has been removed as now backups are stored internally using .tar.gz format by default, you can
  27    set $CFG->usezipbackups to store them in zip format. This does not affect the restore process, which continues accepting both.
  28  * Added support for custom string manager implementations via $CFG->customstringmanager
  29    directive in the config.php. See MDL-49361 for details.
  30  * Add new make_request_directory() for creation of per-request files.
  31  * Added generate_image_thumbnail_from_string. This should be used instead of generate_image_thumbnail when the source is a string.
  32    This prevents the need to write files to disk unnecessarily.
  33  * Added generate_image_thumbnail to stored_file class. This should be used when generating thumbnails for stored files.
  34    This prevents the need to write files to disk unnecessarily.
  35  * Removed pear/HTTP/WebDav. See MDL-49534 for details.
  36  * Use standard PHP date time classes and methods - see new core_date class for timezone normalisation methods.
  37  * Moved lib/google/Google/ to lib/google/src/Google. This is to address autoloader issues with Google's provided autoloader
  38    for the library. See MDL-49519 for details.
  39  * The outdated lib/google/Google_Client.php and related files have been completely removed. To use
  40    the new client, read lib/google/readme_moodle.txt, please.
  41  * profile_display_badges() has been deprecated. See MDL-48935 for details.
  42  * Added a new method add_report_nodes() to pagelib.php. If you are looking to add links to the user profile page under the heading "Reports"
  43    then please use this function to ensure that the breadcrumb and navigation block are created properly for all user profile pages.
  44  * process_new_icon() now does not always return a PNG file. When possible, it will try to keep the format of the original file.
  45    Set the new argument $preferpng to true to force PNG. See MDL-46763 and MDL-50041 for details.

  47  === 2.8 ===
  48  
  49  * Gradebook grade category option "aggregatesubcats" has been removed completely.
  50    This means that the database column is removed, the admin settings are removed and
  51    the properties from the grade_category object have been removed. If any courses were
  52    found to be using this setting, a warning to check the grades will be shown in the
  53    course grader report after upgrading the site. The same warning will be shown on
  54    courses restored from backup that had this setting enabled (see MDL-47503).
  55  * lib/excelllib.class.php has been updated. The class MoodleExcelWorkbook will now only produce excel 2007 files.
  56  * renderers: We now remove the suffix _renderable when looking for a render method for a renderable.
  57    If you have a renderable class named like "blah_renderable" and have a method on a renderer named "render_blah_renderable"
  58    you will need to change the name of your render method to "render_blah" instead, as renderable at the end is no longer accepted.
  59  * New functions get_course_and_cm_from_cmid($cmorid, $modulename) and
  60    get_course_and_cm_from_instance($instanceorid, $modulename) can be used to
  61    more efficiently load these basic data objects at the start of a script.
  62  * New function cm_info::create($cm) can be used when you need a cm_info
  63    object, but have a $cm which might only be a standard database record.
  64  * $CFG->enablegroupmembersonly no longer exists.
  65  * Scheduled tasks have gained support for syntax to introduce variability when a
  66    task will run across installs. When a when hour or minute are defined as 'R'
  67    they will be installed with a random hour/minute value.
  68  * Several classes grade_edit_tree_column_xxx were removed since grades setup page
  69    has been significantly changed. These classes should not be used outside of
  70    gradebook or developers can copy them into their plugins from 2.7 branch.
  71  * Google APIs Client Library (lib/google/) has been upgraded to 1.0.5-beta and
  72    API has changed dramatically without backward compatibility. Any code accessing
  73    it must be amended. It does not apply to lib/googleapi.php. See MDL-47297
  74  * Added an extra parameter to the function get_formatted_help_string() (default null) which is used to specify
  75    additional string parameters.
  76  * User settings node and course node in navigation now support callbacks from admin tools.
  77  * grade_get_grades() optional parameteres $itemtype, $itemmodule, $iteminstance are now required.

 205  === 2.6 ===
 206  
 207  * Use new methods from core_component class instead of get_core_subsystems(), get_plugin_types(),
 208    get_plugin_list(), get_plugin_list_with_class(), get_plugin_directory(), normalize_component(),
 209    get_component_directory() and get_plugin_list_with_file(). The names of the new methods are
 210    exactly the same, the only differences are that core_component::get_plugin_types() now always returns
 211    full paths and core_component::get_plugin_list() does not accept empty parameter any more.
 212  * Use core_text::* instead of textlib:: and also core_collator::* instead of collatorlib::*.
 213  * Use new function moodleform::mock_submit() to simulate form submission in unit tests (backported).
 214  * New $CFG->localcachedir setting useful for cluster nodes. Admins have to update X-Sendfile aliases if used.
 215  * MS SQL Server drivers are now using NVARCHAR(MAX) instead of NTEXT and VARBINARY(MAX) instead of IMAGE,
 216    this change should be fully transparent and it should help significantly with add-on compatibility.
 217  * The string manager classes were renamed. Note that they should not be modified or used directly,
 218    always use get_string_manager() to get instance of the string manager.
 219  * The ability to use an 'insecure' rc4encrypt/rc4decrypt key has been removed.
 220  * Use $CFG->debugdeveloper instead of debugging('', DEBUG_DEVELOPER).
 221  * Use set_debugging(DEBUG_xxx) when changing debugging level for current request.
 222  * Function moveto_module() does not modify $mod argument and instead now returns the new module visibility value.
 223  * Use behat_selectors::get_allowed_text_selectors() and behat_selectors::get_allowed_selectors() instead of
 224    behat_command::$allowedtextselectors and behat_command::$allowedselectors
 225  * Subplugins are supported in admin tools and local plugins.
 226  * file_packer/zip_packer API has been modified so that key functions support a new file_progress interface
 227    to report progress during long operations. Related to this, zip_archive now supports an estimated_count()
 228    function that returns an approximate number of entries in the zip faster than the count() function.
 229  * Class cm_info no longer extends stdClass. All properties are read-only and calculated on first request only.
 230  * Class course_modinfo no longer extends stdClass. All properties are read-only.
 231  * Database fields modinfo and sectioncache in table course are removed. Application cache core/coursemodinfo
 232    is used instead. Course cache is still reset, rebuilt and retrieved using function rebuild_course_cache() and
 233    get_fast_modinfo(). Purging all caches and every core upgrade purges course modinfo cache as well.
 234    If function get_fast_modinfo() is called for multiple courses make sure to include field cacherev in course
 235    object.
 236  * Internal (noreply and support) user support has been added for sending/receiving message.
 237    Use core_user::get_noreply_user() and core_user::get_support_user() to get noreply and support user's respectively.
 238    Real users can be used as noreply/support users by setting $CFG->noreplyuserid and $CFG->supportuserid
 239  * New function readfile_allow_large() in filelib.php for use when very large files may need sending to user.
 240  * Use core_plugin_manager::reset_caches() when changing visibility of plugins.
 241  * Implement new method get_enabled_plugins() method in subplugin info classes.
 242  * Each plugin should include version information in version.php.
 243  * Module and block tables do not contain version column any more, use get_config('xx_yy', 'version') instead.
 244  * $USER->password field is intentionally unset so that session data does not contain password hashes.
 245  * Use core_shutdown_manager::register_function() instead of register_shutdown_function().
 246  * New file packer for .tar.gz files; obtain by calling get_file_packer('application/x-gzip'). Intended initially
 247    for use in backup/restore only, as there are limitations on supported filenames. Also new packer for
 248    backups which supports both compression formats; get_file_packer('application/vnd.moodle.backup').
 249  * New optional parameter to stored_file::get_content_file_handle to open file handle with 'gzopen' instead
 250    of 'fopen' to read gzip-compressed files if required.
 251  * update_internal_user_password() and setnew_password_and_mail() now trigger user_updated event.
 252  * Add thirdpartylibs.xml file to plugins that bundle any 3rd party libraries.
 253  * New class introduced to help auto generate zIndex values for modal dialogues. Class "moodle-has-zindex"
 254    should set on any element which uses a non-default zindex and needs to ensure it doesn't show above a
 255    dialogue.
 256  * $CFG->filelifetime is now used consistently for most file serving operations, the default was lowered
 257    to 6 hours from 24 hours because etags and x-sendfile support should make file serving less expensive.
 258  * Date format locale charset for windows server will come from calendar type and for gregorian it will use
 259    lang file.
 260  * The library to interact with Box.net (class boxclient) is only compatible with their APIv1 which
 261    reaches its end of life on the 14th of Dec. You should migrate your scripts to make usage of the
 262    new class boxnet_client(). Note that the method names and return values have changed.
 263  * Settings pages are now possible for Calendar type plugins. Calendar type plugins that require a settings page to
 264    work properly will need to set their requires version to a number that is equal to or grater than the 2.6.1 release version.
 265  * The admin/tool/generator tool was overhauled to use testing data generators and the previous interface to create
 266    test data was removed (it was not working correctly anyway). If you were using this tool you will probably need to
 267    update your code.
