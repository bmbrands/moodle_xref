   1  This files describes API changes for course formats
   2  
   3  Overview of this plugin type at http://docs.moodle.org/dev/Course_formats
   4  
   5  === 2.9 ===
   6  * Course formats may support deleting sections, see MDL-10405 for more details.
   7    format_section_renderer_base::section_edit_controls() is now also called for
   8    stealth sections and it also returns "delete" control.
   9  
  10  === 2.8 ===
  11  * The activity chooser now uses M.course.format.get_sectionwrapperclass()
  12    to determine the section selector, rather than a hard-coded `li.section`.
  13  * Activity duplication in /course/modduplicate.php is deprecated and is now done in /course/mod.php.  Deprecated calls will be honored by
  14    redirecting to /course/mod.php for 3rd party plugin support.
  15  * New method format_base::section_get_available_hook() allows plugins to override section availability.
  16  
  17  === 2.7 ===
  18  * The ->testedbrowsers array no longer needs to be defined in supports_ajax().
  19  * format_section_renderer_base::section_hidden has an new second optional argument $courseorid.
  20    If provided the section name is printed in the not available string when displaying the hidden section.
  21  
  22  === 2.6 ===
  23  
  24  * core_course_renderer::course_section_cm_edit_actions has two new optional arguments and now uses and action_menu component.
  25  * core_course_renderer::course_section_cm has been altered to call core_course_renderer::course_section_cm_edit_actions with the two new arguments
  26  * An additional course renderer function has been created which allows you to
  27    specify the wrapper for a course module within a section (e.g. the <li>).  This can be
  28    found in core_course_renderer::course_section_cm_list_item().
  29  
  30  === 2.5 ===
  31  
  32  * Functions responsible for output in course/lib.php are deprecated, the code is moved to
  33    appropriate renderers: print_section(), print_section_add_menus(), get_print_section_cm_text(),
  34    make_editing_buttons()
  35    See functions' phpdocs in lib/deprecatedlib.php
  36  * Function get_print_section_cm_text() is deprecated, replaced with methods in cm_info
  37  
  38  === 2.4 ===
  39  
  40  Course format API has been changed significantly. Instead of implementing callbacks course formats
  41  may overwrite the class format_base. See format_legacy class for a template for upgrading course
  42  format.
  43  
  44  * Function settings_navigation::add_course_editing_links() is completely removed, course format
  45    functions callback_XXXX_request_key() are no longer used (where XXXX is the course format name)
  46  * functions get_generic_section_name(), get_all_sections(), add_mod_to_section(), get_all_mods()
  47    are deprecated. See their phpdocs in lib/deprecatedlib.php on how to replace them
  48  * Course formats may now have their settings.php file as the most of other plugin types
  49  * Function format_section_renderer_base::is_section_current() is deprecated, overwrite/use
  50    function is_section_current in format class
  51  
  52  === 2.3 ===
  53  
  54  * The new $course->coursedisplay option was introduced, users can now choose to display
  55    a section at a time if the course formats support it:
  56   - COURSE_DISPLAY_SINGLEPAGE indicates the teacher has chosen to display all sections on one page
  57   - COURSE_DISPLAY_MULTIPAGE indicates the teacher has chose to have seperate pages with each section.
  58  
  59  * The parameter for 'currently active section' was standardised in core:
  60    - The course format is passed the currently live section through the $displaysection varaible to format.php
  61    - A 'section' paramter is the standardised way to pass around the current section in a course
  62    - Navigation no longer looks for custom parameters defined in callback_format_request_key
