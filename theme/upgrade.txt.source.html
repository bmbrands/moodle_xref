   1  This files describes API changes in /theme/* themes,
   2  information provided here is intended especially for theme designer.
   3  
   4  === 2.9 ===
   5  
   6  * Themes Bootstrapbase, Clean and More have undergone some changes for RTL layouts see - MDL-48160.
   7  
   8  * Themes Clean amd More page header background logo, if loaded, no longer links to the home page.
   9  * Themes Clean and More have been updated to use the renderer full_header(). This renderer (when in a user context) displays a message button
  10    and consequently the link to message users have been removed from various pages. It is recommended that theme developers use this
  11    renderer to ensure that they do not suffer from a reduction in access to this feature.
  12  * $OUTPUT->context_header() has been introduced to produce headers similar to what full_header() creates. This renderer is used for creating
  13    subsequent headers in a page. If provided with user information, this render will also display a message button like full_header().
  14  
  15  === 2.8 ===
  16  
  17  * Themes Base and Canvas layout/embedded.php has changed.
  18    The div id="content" has been changed to id="page-content" - see MDL-47915.
  19  * All mod_assign CSS selectors have been qualified with ".path-mod-assign". This means any custom CSS
  20    rules for mod_assign in a theme may need to be updated.
  21  * The following render methods were renamed, if you have overridden any of these you will need to update your renderer:
  22    - report_log_renderer::render_report_log_renderable         => report_log_renderer::render_report_log
  23    - report_loglive_renderer::render_report_loglive_renderable => report_loglive_renderer::render_report_log
  24  * On calendar view screen the course selector and new event button have been swapped, custom CSS for .path-calendar might be affected.
  25  * The functions search_listing() and search_pagination() in the core_course_management_renderer class take the searched string
  26    as an extra parameter - see MDL-46902.
  27  * Themes with non-fixed headers must have the .navbar class in their navbar in order for floating headers in the grader report to work - see MDL-46658.
  28  * To accomodate the new gradebook, use of overflow: hidden or overflow-x: hidden CSS rules on DOM elements containing the
  29    #user-grades table, visible at [Course administration -> Grades] is not recommended. Please use overflow: auto or
  30    overflow: scroll instead, as this will handle medium to large courses with multiple activities better (more information available at https://docs.moodle.org/dev/Theme_upgrade_notes ).
  31  * The new user menu can be added to themes by calling $OUTPUT->user_menu(); we recommend doing so in a backwards-compatible way (see https://docs.moodle.org/dev/Theme_upgrade_notes for more information).
  32  
  33  === 2.7 ===
  34  
  35  * CSS related functions in theme class and csslib.php were refactored, addons that are
  36    using this private API need to be updated
  37  * Please update your css to use 'tr:nth-of-type(odd/even)' instead of '.r0/.r1' to apply tr specific css for various tables.
  38    These classes are deprecated now and will be removed in Moodle 2.9. This has done to better support styling tables that will be altered by JavaScript.
  39    Here is an example to update your css:
  40    Old code:
  41      table.generaltable .r0 {background-color: #F0F0F0;}
  42      table.generaltable .r1 {background-color: #F0F0F0;}
  43    New code:
  44      table.generaltable > tbody > tr:nth-of-type(even) {background-color: #F0F0F0;}
  45      table.generaltable > tbody > tr:nth-of-type(odd) {background-color: #F0F0F0;}
  46  
  47  Removed themes:
  48  * afterburner, anomaly, arialist, binarius, boxxie, brick, formal_white, formfactor, fusion, leatherbound,
  49    magazine, nimble, nonzero, overlay, serenity, sky_high, splash, standard and standardold have been removed.
  50    During the upgrade process the themes will be uninstalled and all their settings will be deleted.
  51    If you wish to keep your theme and its settings, download it from moodle.org and put it back in
  52    the theme/ directory BEFORE UPGRADING.
  53  
  54  Notes:
  55  * z-index on #dock has been adjusted from 11000 to 5000 to prevent overlapping issues with fullscreen TinyMCE in Base theme.

  57  === 2.6 ===
  58  
  59  Removed themes:
  60  * "My Mobile" theme has been removed (see MDL-40874).
  61    During the upgrade process "My Mobile" theme will be uninstalled and all its settings will be deleted.
  62    "My Mobile" theme and its extending themes will fallback to the mobile friendly "Clean" theme.
  63    If you wish to keep "My Mobile" theme and its settings, BEFORE UPGRADING you can copy the files from the
  64    unmaintained plugin into theme/mymobile.
  65    The unmaintained plugin can be downloaded at https://moodle.org/plugins/pluginversion.php?id=4563
  66  Notes:
  67  * There have been several changes made to the classes used on headings throughout Moodle.
  68    This was done as per the Moodle HTML guidelines. h1 and h2 elements should have no classes, h3 and lower headings
  69    may have descriptive classes only (any classes must properly describe the relation between the header and its content).
  70  * The parameters value for the following functions have been modified: core_renderer::heading() and core_renderer::heading_with_help().
  71    This means that headings using the default options will no longer have the class "main" which was the previous default.
  72  * The file picker and activity chooser have been converted to use M.core.dialogue. This means the styles
  73    for these elements have changed in the base and bootstrapbase themes and you will need to copy the new styles
  74    if you are not inheriting from either one of these themes.
  75  * For the themes based on bootstrapbase please also read theme/bootstrapbase/upgrade.txt
  76  * A new component action_menu is now used to display editing icons for courses and blocks within a drop down. If you have a theme that doesn't
  77    extend base, canvas, or clean then you will need to style for this new component within your theme.
  78  * The maintenance layout now has a special renderer that extends the core_renderer in order to prevent some methods from interacting
  79    with the database. Please be aware that for the maintenance layout some methods now always return an empty string.
  80    This has been done because it is important that during maintenance routines we don't show any links, interact with the database,
  81    or interact with caches as doing so may lead to errors,
  82    Please see the maintenance renderer notes below for details on the functions affected.
  83  
  84  Renderer changes:
  85  * core_course_renderer::course_section_cm_edit_actions has two new optional arguments and now uses and action_menu component.
  86  * core_course_renderer::course_section_cm has been altered to call core_course_renderer::course_section_cm_edit_actions with the two new arguments.
  87  * core_renderer::block_controls has a new optional argument (the block id) and has been altered to use the new action_menu component.
  88  * core_renderer::block_header has been altered to give core_renderer::block_controls the new second argument, the block id.
  89  * core_renderer::action_link has a new argument, and optional icon that can be rendererd into the link.
  90  
  91  Selector changes:
  92  * Changes to the classes given to the header produced when rendering a single section within a course.
  93    .section-navigation.header.headingblock => .section-navigation.navigationtitle
  94  * Changes to the classes used when producing the course information box that lists courses.
  95    The name (h3|div} of both courses and categories used to have .name, for courses in the list this was changed to .coursename and for categories in the list this was changed to .categoryname.
  96  * Classes ".header and .headingblock" were removed from all front page content headings.
  97  * Classes ".headingblock .header .tag-header" were removed from the tag index page
  98  
  99  Maintenance renderer notes:
 100  When the maintenance layout is being used $OUTPUT will be an instance of core_renderer_maintenance.
 101  This renderer mimics the core_renderer except that the following functions always return an empty string.
 102  * core_renderer_maintenance::block

 104  * core_renderer_maintenance::blocks_for_regions
 105  * core_renderer_maintenance::course_header
 106  * core_renderer_maintenance::course_footer
 107  * core_renderer_maintenance::course_content_header
 108  * core_renderer_maintenance::course_content_footer
 109  * core_renderer_maintenance::custom_menu
 110  * core_renderer_maintenance::file_picker
 111  * core_renderer_maintenance::htmllize_file_tree
 112  * core_renderer_maintenance::lang_menu
 113  * core_renderer_maintenance::login_info
 114  * core_renderer_maintenance::user_picture
 115  
 116  If you have overridden methods of the core_renderer in your theme and want those changes to be shown during times of maintenance you
 117  will also need to override the core_renderer_maintenance and copy your customisations from the core_renderer to that.

 119  === 2.5.1 ===
 120  
 121  Notes:
 122  * Block drag and drop functionality has been improved so that it works regardless of what block regions you use
 123    or how many block regions you have. In order to benefit from this improvement you must convert your calls from
 124    $OUTPUT->blocks_for_region() to $OUTPUT->blocks.
 125  
 126  Optional changes:
 127  * Theme config can nominate block regions to swap if the user is using a rtl languages such as Hebrew.
 128    $THEME->blockrtlmanipulations should be an associative array, the key is the original block region, and the
 129    value should be where the blocks in that region should be displayed.
 130  * New $OUTPUT methods to replace code that was previously using $PAGE. Converting to these methods is optional
 131    but highly recommended. Refer to the bootstrapbase layouts for examples.
 132    - $OUTPUT->body_attributes() returns a string you can use for the body id and classes.
 133    - $OUTPUT->blocks() returns HTML to display a block region and all of its blocks. This adds data attributes
 134      that ensure drag and drop of blocks will work no matter what block regions or how many you have.
 135    - $OUTPUT->page_heading() returns the page heading. Replaces $PAGE->heading.
 136    - $OUTPUT->page_heading_button() returns the button to display with the heading. Replaces $page->button.
 137    - $OUTPUT->page_doc_link() returns the link to moodle docs for the page. Replaces page_doc_link().
 138    - $OUTPUT->page_heading_menu() returns the heading menu for the page. Replaces $PAGE->headingmenu.
 139    - $OUTPUT->page_title() Returns the title to use for the head section.
 140    - $OUTPUT->favicon() returns the URL to the favicon.
 141  
 142  Renderer changes:
 143  * core_renderer::navbar now returns an empty string if there are no navigation items to display.
 144  * core_renderer::custom_menu now adds a class "custom_menu" to the div that contains the HTML for the custom menu.
 145  
 146  === 2.5 ===
 147  
 148  required changes:
 149  * Functions core_course_renderer::course_category_tree() and course_category_tree_category()
 150    are deprecated
 151  * Significant changes in rendering of courses and categories listings, lots of CSS classes changed,
 152    several functions such as print_courses(), print_whole_category_list(), print_category_info()
 153    are moved to course renderer.
 154    See http://docs.moodle.org/dev/Courses_lists_upgrade_to_2.5

 156  DOM changes:

 158  * new setting $THEME->yuicssmodules = array('cssreset', 'cssfonts', 'cssgrids', 'cssbase'); which
 159    allows themes to use different CSS reset normalisers such as cssnormalize YUI module.
 160  * Re-wrote the user profile views to definition lists.
 161  * Re-wrote the table for the course completion status block to use html_table - added some CSS classes to
 162    the table in the process (see MDL-35608).
 163  * Cancel buttons have the class btn-cancel.
 164  * Added a z-index for the div#dateselector-calendar-panel so that the calendar pop-up renders above
 165    the filemanager when they overlap, ie. the course settings page (see MDL-39047).
 166  * Trivial change of CSS selectors used to display plugin status at the plugins overview

 169  Renderer changes:
